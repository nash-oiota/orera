# NFR Requirements — Unit 2: agent-configs

## セキュリティ（Security Baseline 適用）

| ルール | 適用 | 対応 |
|---|---|---|
| SECURITY-06 | ✅ | `allowedTools` に必要最小限のツールのみ列挙。初期は網羅的に設定し運用で絞る |
| SECURITY-09 | ✅ | JSON ファイルにシークレット・APIキー・個人情報をハードコードしない |
| SECURITY-11 | ✅ | PdM・specialist の役割を明確に分離し、責務の混在を防ぐ |
| SECURITY-15 | ✅ | PdM プロンプトに「エラー・例外時は BLOCKED で報告する」ルールを明記 |
| SECURITY-01〜05,07,08,10,12,13,14 | N/A | ローカルツール・ネットワーク露出なし・認証不要 |

## 信頼性

- specialist が指示に従わない場合（プロンプト無視）は MVP として許容し、手動介入で対処する
- PdM プロンプトに自律判断の境界を明確に定義し、予測不能な動作を最小化する

## 保守性

- 各エージェントの JSON は独立したファイルで管理し、1ファイル = 1エージェントを維持する
- プロンプトは JSON の `prompt` フィールドに直接記述する（外部ファイル参照なし）
- チーム構成の変更は JSON ファイルの追加・削除のみで対応できる設計を維持する
