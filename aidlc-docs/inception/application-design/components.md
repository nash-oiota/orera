# Components — kiro-team

## config.sh
- **種別**: 設定ファイル（Shell）
- **責務**: セッション名・ディレクトリパス・ポーリング間隔などの共通設定を一元管理する
- **インターフェース**: 各スクリプトが `source scripts/config.sh` で読み込む

## start.sh
- **種別**: スクリプト
- **責務**: PdM tmux セッションと watcher.sh・notifier.sh を起動する。tasks/ と results/ ディレクトリを初期化する
- **インターフェース**: ユーザーが手動実行

## stop.sh
- **種別**: スクリプト
- **責務**: kiro-team tmux セッション配下の全ウィンドウを停止する
- **インターフェース**: ユーザーが手動実行

## status.sh
- **種別**: スクリプト
- **責務**: 全エージェントの tmux セッション生死と results/*.md の最終更新時刻を表示する
- **インターフェース**: ユーザーが手動実行 / PdM が execute_bash 経由で実行

## watcher.sh
- **種別**: 常駐スクリプト
- **責務**: results/ を定期ポーリングし、未通知の変更を検知かつ PdM がプロンプト待ちの場合のみ tmux send-keys で通知する。通知済みマーク（.notified）で重複通知を防ぐ
- **インターフェース**: start.sh から起動される tmux ウィンドウで常駐

## notifier.sh
- **種別**: 常駐スクリプト
- **責務**: tasks/ を定期ポーリングし、未送信タスクを検知かつ対象 specialist がプロンプト待ちの場合のみ tmux send-keys でタスクを送信する。送信済みマーク（.sent）で重複送信を防ぐ
- **インターフェース**: start.sh から起動される tmux ウィンドウで常駐

## pdm.json
- **種別**: kiro-cli エージェント設定
- **責務**: PdM エージェントの定義。ユーザーからの指示を受け取り、タスク分解・委任・自律判断・マネジメントを行う
- **インターフェース**: kiro-cli が読み込む JSON 設定ファイル

## reviewer.json
- **種別**: kiro-cli エージェント設定
- **責務**: frontend と backend の結合観点でコードレビューを行う。API契約・データフロー・セキュリティの統合問題を検出する
- **インターフェース**: kiro-cli が読み込む JSON 設定ファイル
- **種別**: kiro-cli エージェント設定
- **責務**: 各専門領域のスペシャリストエージェントの定義。tasks/ からタスクを受け取り、実行後に results/ に結果を書き込む
- **インターフェース**: kiro-cli が読み込む JSON 設定ファイル
